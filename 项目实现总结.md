# 视频到笔记 — 项目实现总结

> 输入视频链接 → 输出结构化 Markdown 笔记，全链路自动化

---

## 一、项目概述

### 目标
打造一个零门槛的视频笔记工具：复制链接 → 回车 → 拿到笔记，不依赖任何付费 API。

### 核心链路

```
URL → yt-dlp 下载字幕
      ├─ 有字幕 → 解析纯文本 ✅（秒级）
      └─ 无字幕 → 音频下载 → ffmpeg 转 WAV → FunASR 转文字 🔉（分钟级）
                                          ↓
                                Claude Code 生成笔记
                                          ↓
                                    📝 笔记.md
```

### 技术选型

| 环节 | 工具 | 选择理由 |
|------|------|----------|
| 下载 | yt-dlp (python -m) | 覆盖 1800+ 站点，自动字幕抽取 |
| 转码 | ffmpeg | 工业标准，winget 一条命令安装 |
| 语音转文字 | FunASR paraformer-zh | 阿里达摩院开源，通义千问同款引擎，无限免费 |
| AI 总结 | Claude Code --print | 已有订阅，非交互模式可直接管道调用 |
| 脚本 | PowerShell 5.1 | Windows 原生，用户零额外环境配置 |
| JS 运行时 | deno | yt-dlp 解 YouTube JS 挑战必需 |

## 三、关键设计决策

### 3.1 为什么用 python -m yt_dlp 而不是 yt-dlp 命令

**问题**：pip 安装 yt-dlp 后，可执行文件在 `Python\Scripts\` 目录，但该目录不一定在 PATH 中（尤其是 PowerShell 7）。

**解决**：脚本启动时从注册表刷新 PATH，同时扫描 winget 安装目录。yt-dlp 统一使用 `python -m yt_dlp` 调用，彻底避免 PATH 问题。

### 3.2 为什么脚本必须带 UTF-8 BOM

**问题**：PowerShell 5.1 读取无 BOM 的文件时，默认使用系统 ANSI 代码页（中文 Windows 为 GBK）。UTF-8 编码的中文注释/字符串会被解析为乱码，导致 `MB`、`KB` 等合法 token 被错误拆分，引发语法错误。

**解决**：脚本文件必须以 UTF-8 with BOM (`EF BB BF`) 保存。`Write` 工具默认写无 BOM 的 UTF-8，需要用 `[System.IO.File]::WriteAllText(path, content, [System.Text.UTF8Encoding]::new($true))` 显式加上 BOM。

### 3.3 PowerShell 的 $ErrorActionPreference 陷阱

**问题**：`$ErrorActionPreference = "Stop"` 会导致 PowerShell 把**任何**写入 stderr 的输出都当作错误。Python/FunASR 的所有进度信息（加载模型、下载进度、推理统计）都走 stderr，直接触发脚本中止。

**解决**：调用 Python 前临时切换到 `Continue` 模式，调用后恢复，只通过 `$LASTEXITCODE` 判断真伪：
```powershell
$savedEAP = $ErrorActionPreference
$ErrorActionPreference = "Continue"
$output = & python script.py   # stderr 直出终端，stdout 被捕获
$ErrorActionPreference = $savedEAP
if ($LASTEXITCODE -ne 0) { throw "real error" }
```

### 3.4 Claude Code 管道编码问题

**问题**：PowerShell 管道传中文到外部命令时，默认使用系统编码（GBK），导致 Claude 收到乱码。

**解决**：将 prompt 写入临时 UTF-8 文件，通过 `cmd /c chcp 65001 & type file | claude --print` 传入。CMD 的 `type` 命令直接输出字节流，不经过 PowerShell 编码转换。

### 3.5 长音频 OOM（内存溢出）

**问题**：FunASR paraformer 的自注意力机制是 O(n²) 内存复杂度。16kHz 采样率下，15 分钟音频 ≈ 14M 采样点，注意力矩阵约需 3.7GB 内存，超出可用内存。

**解决**：asr.py 自动检测音频时长，超过 3.5 分钟自动切分为 3 分钟段，逐段转写后合并。单段内存需求降至 ~150MB。实现：
```python
chunk_sec = 180  # 3 分钟/段
if duration_sec > chunk_sec + 30:
    # ffmpeg 切分 → 逐段 model.generate() → 合并结果
```

### 3.6 B站 反爬（HTTP 412）

**问题**：B站 API 要求 WBI 签名 + 登录态。无 Cookie 时永远 412。

**方案对比**：

| 方案 | 可行性 | 问题 |
|------|--------|------|
| 模拟移动端 UA | ❌ | 仍然 412 |
| `--cookies-from-browser` | ⚠️ | 浏览器运行时 SQLite 被锁 |
| cookies.txt 文件 | ✅ | 一次性导出，永久使用 |

**最终方案**：用户通过浏览器 F12 控制台执行 `copy(document.cookie...)` 导出 Cookie 到 `bilibili_cookies.txt`，脚本检测到文件路径时自动使用 `--cookies` 参数。

### 3.7 FunASR 多模型加载问题

**初始设计**：同时加载 4 个模型（paraformer + VAD + 标点 + 说话人分离），一次性下载量巨大，下载超时导致失败。

**最终设计**：只加载 paraformer 核心模型。VAD 和标点恢复已内置于 paraformer-zh 的预处理中。模型首次下载 ~944MB，之后永久缓存到 `~/.cache/modelscope/`。

### 3.8 Claude 输出中文乱码

**问题**：脚本通过 `cmd /c "chcp 65001 & type prompt.txt | claude --print"` 获取 Claude 返回的 UTF-8 字节流。但 PowerShell 的 `[Console]::OutputEncoding` 默认为系统 ANSI 编码（中文 Windows 为 GBK/cp936），捕获 stdout 时用 GBK 解码 UTF-8，导致所有中文字符变成乱码。即使后续用 UTF-8 写入文件，内容已经损坏。

**解决**：调用前设置 `[Console]::OutputEncoding = [System.Text.Encoding]::UTF8`，调用后在 `finally` 块中恢复：
```powershell
$prevEncoding = [Console]::OutputEncoding
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
try {
    $claudeOutput = cmd /c "chcp 65001 >nul & type `"$promptFile`" | claude --print --output-format text" 2>&1
} finally {
    [Console]::OutputEncoding = $prevEncoding
}
```
> 这个坑比较隐蔽：cmd 侧设置了 `chcp 65001` 保证**发送**的是 UTF-8，但 PowerShell **接收**时用自己的编码解码。两端编码必须一致。

## 五、数据流

```
用户输入: .\video2note.ps1 "URL"

1. PATH 刷新
   ├─ 读注册表 Machine/User PATH
   └─ 扫描 winget 安装目录

2. 获取视频信息
   ├─ python -m yt_dlp --print "%(title)s" --no-download URL
   └─ 输出: 视频标题、安全文件名

3. 尝试下载字幕
   ├─ yt-dlp --write-auto-subs --sub-langs "zh-Hans,zh-CN,zh,en,en-US" --skip-download
   ├─ 搜索 *.srt + *.vtt 文件
   ├─ 有 → Parse-SrtToText() 解析 → 纯文本 → 跳至步骤 5
   └─ 无 → 进入步骤 4

4. 语音转文字
   ├─ yt-dlp -x --audio-format mp3 → audio.mp3
   ├─ ffmpeg -ar 16000 -ac 1 -acodec pcm_s16le → audio_16k.wav
   ├─ python asr.py audio_16k.wav
   │   ├─ 加载 paraformer-zh 模型（首次 ~5min 下载，之后秒加载）
   │   ├─ 时长 ≤ 3.5min → 一次性 model.generate(batch_size_s=300)
   │   └─ 时长 > 3.5min → ffmpeg 切为 3min×N 段 → 逐段转写 → 合并
   └─ stdout: 转写文本

5. Claude Code 生成笔记
   ├─ 模板填充: {title} + {url} + {text}
   ├─ 写入 UTF-8 临时文件 prompt.txt
   ├─ cmd /c "chcp 65001 & type prompt.txt | claude --print --output-format text"
   └─ 输出: Markdown 笔记

6. 保存 & 清理
   ├─ 笔记 → notes/标题.md (UTF-8 无 BOM)
   └─ 清理临时文件（除非 -NoCleanup）
```

## 六、踩坑记录

### 6.1 PowerShell 字符串中的 KB/MB 陷阱
变量名以 `KB`、`MB`、`GB` 结尾时（如 `$sizeMB`），在双引号字符串中会被 PowerShell 解析器误认为数字常量后缀，拆分为 `$size` × `MB`。**教训**：变量名避免以计量单位结尾，或用 `$($var)` 强制隔离。

### 6.2 ffmpeg -sample_fmt vs -acodec
`ffmpeg -sample_fmt s16` 对 WAV 容器不兼容（WAV 要求 `s16le`）。正确写法：`-acodec pcm_s16le`。

### 6.3 FunASR 模型命名
- 短名 `paraformer-zh` 可用（AutoModel 自动解析为完整 ModelScope ID）
- 但 `paraformer-zh` 实际上是 `seaco_paraformer` 变体，内存消耗比普通版大
- 如需更低内存可考虑 `iic/speech_paraformer_asr_nat-zh-cn-16k-common-vocab8404-pytorch`

### 6.4 YouTube 字幕下载 429
yt-dlp 批量下载字幕时，如果指定了多个语言（zh-Hans, en...），第一个语言被限流（429）会导致整个字幕下载失败。**改进方向**：逐个语言尝试，单个失败不影响其他。

## 七、可继承的模块

### 7.1 Parse-SrtToText (PowerShell)
通用的字幕文件（SRT/VTT）解析器，去除时间戳、行号、HTML 标签，输出纯文本。可直接复用到任何需要字幕提取的项目。

### 7.2 asr.py (Python)
独立的语音转文字模块，输入 16kHz mono WAV，输出纯文本。自动分段处理长音频。依赖只有 `funasr`，可单独使用。

### 7.3 PATH 刷新代码段 (PowerShell)
```powershell
$env:PATH = [System.Environment]::GetEnvironmentVariable("PATH", "Machine") + ";" + [System.Environment]::GetEnvironmentVariable("PATH", "User")
$wingetPackages = "$env:LOCALAPPDATA\Microsoft\WinGet\Packages"
if (Test-Path $wingetPackages) {
    Get-ChildItem $wingetPackages -Directory | ForEach-Object {
        if (Test-Path "$($_.FullName)\*.exe") { $env:PATH = "$($_.FullName);$env:PATH" }
    }
}
```
适用于所有 winget 安装工具的 Windows 脚本项目。

### 7.4 CMD 管道 UTF-8 编码方案
```powershell
[System.IO.File]::WriteAllText($file, $content, [System.Text.UTF8Encoding]::new($false))
$output = cmd /c "chcp 65001 >nul & type `"$file`" | some-command"
```
适用于所有需要从 PowerShell 管道 UTF-8 中文到外部命令的场景。

## 八、改进方向

1. **字幕下载容错**：逐个语言尝试下载，单个失败不影响其他语言
2. **更轻量的 ASR 模型**：`paraformer-zh-small` 模型更小（~200MB），适合内存紧张环境
3. **GPU 加速**：FunASR 支持 CUDA，安装 PyTorch CUDA 版可大幅提速（当前 CPU 推理 12秒/3分钟段）
4. **笔记格式可选**：支持纯文本、大纲、思维导图等输出格式
5. **进度条整合**：Python 的 tqdm 进度条目前直出终端，可以解析后集成到 PowerShell Write-Progress
6. **错误恢复**：断点续传，避免重新下载音频
