# 视频到笔记 — 使用教程

> 视频链接 / 音频文件 / 视频文件 → 一键输出结构化 Markdown 笔记（精炼版 + 详细版）
> 自带图形界面，也支持命令行

---

## 快速上手（GUI）

```powershell
python gui.py
```

1. 选择输入模式：**视频链接** / **音频文件** / **视频文件**
2. 填入链接或选择文件
3. 点击 **▶ 开始**

处理完成自动生成两份笔记：精炼版（结构总结）+ 详细版（完整记录）。

## 前置准备

### 第一步：安装依赖

```powershell
# 1. yt-dlp（视频下载，覆盖 1800+ 站点）
pip install yt-dlp

# 2. ffmpeg（音频格式转换）
winget install ffmpeg

# 3. FunASR（语音转文字，通义千问同款引擎）
pip install funasr

# 4. deno（YouTube JS 运行时，必须装）
winget install deno
```

> **不需要重启终端**，脚本启动时会自动刷新 PATH。

### 第二步：确认 Claude Code 已登录

```powershell
claude login
```

如果已在使用 Claude Code 则跳过。

### 第三步：B站专属 — 导出 Cookie（一次操作，永久使用）

B站要求登录态，只需操作一次：

1. Edge 打开 B站并**登录**
2. 按 **F12** → 选 **"控制台"**（Console）
3. 粘贴以下代码 → 回车：

```javascript
copy(document.cookie.split('; ').map(c => c.replace('=', '\t') + '\tFALSE\t/\tFALSE\t0\t' + c.split('=')[0]).join('\n'))
```

4. 在项目目录（`f:\00-技术提升\视频到笔记\`）新建文件 `bilibili_cookies.txt`
5. **Ctrl+V** 粘贴 → 保存

> Cookie 文件不要分享给任何人。

## 快速开始

```powershell
cd "f:\00-技术提升\视频到笔记"

# YouTube（直接跑）
.\video2note.ps1 "https://www.youtube.com/watch?v=xxxxx"

# B站（用导出的 cookies.txt）
.\video2note.ps1 "https://www.bilibili.com/video/BVxxxxx" -Cookies ".\bilibili_cookies.txt"
```

### 命令行参数

| 参数 | 必填 | 说明 | 示例 |
|------|------|------|------|
| `-Url` | ✅ | 视频链接 | `-Url "https://..."` |
| `-Cookies` | ❌ | cookies.txt 文件路径，或浏览器名 | `-Cookies ".\bilibili_cookies.txt"` |
| `-OutputDir` | ❌ | 输出目录，默认 `./notes` | `-OutputDir "D:\我的笔记"` |
| `-NoCleanup` | ❌ | 保留中间文件，方便调试 | `-NoCleanup` |

## 常见场景

### 场景一：B站 412 反爬

用 cookies.txt 文件即可：

```powershell
.\video2note.ps1 "https://www.bilibili.com/video/BVxxxxx" -Cookies ".\bilibili_cookies.txt"
```

> 注意：Cookie 有时效性，如果 B站 登录过期需要重新导出。

### 场景二：没有字幕的视频

自动走语音转文字流程：

1. yt-dlp 下载音频 → ffmpeg 转 16kHz WAV
2. FunASR（通义千问同款引擎）语音转文字
   - **短音频（<3分钟）**：直接处理
   - **长音频（>3分钟）**：自动分段，每 3 分钟一段，避免内存溢出
3. Claude Code 生成笔记

**首次使用 FunASR 会自动下载 paraformer-zh 模型（~1GB，约 4-5 分钟）**，之后永久缓存。

### 场景三：调试问题

```powershell
.\video2note.ps1 "https://..." -NoCleanup
```

`notes\_work_<时间戳>\` 目录下保留：
- `video.zh-Hans.srt` / `video.en.vtt` — 字幕文件
- `audio.mp3` — 下载的音频
- `audio_16k.wav` — 转换后的音频（16kHz mono）
- `prompt.txt` — 发送给 Claude 的完整提示词

### 场景四：自定义笔记风格

编辑 `config.json` 的 `notePrompt` 字段，可用变量：
- `{title}` — 视频标题
- `{url}` — 视频链接
- `{text}` — 字幕/转写文本

### 场景五：批量处理

```powershell
$urls = @(
    "https://www.youtube.com/watch?v=xxxxx",
    "https://www.bilibili.com/video/BVyyyyy"
)
foreach ($url in $urls) {
    if ($url -match "bilibili") {
        .\video2note.ps1 -Url $url -Cookies ".\bilibili_cookies.txt"
    } else {
        .\video2note.ps1 -Url $url
    }
}
```

## 工作流程

```
输入链接
  │
  ├─ yt-dlp 下载字幕 (.srt / .vtt)
  │   └─ 有字幕 → 解析纯文本 ✅
  │
  └─ 无字幕 → 下载音频 → ffmpeg 转 16kHz WAV
                └─ FunASR paraformer-zh 语音转文字
                   ├─ 短音频: 一次性处理
                   └─ 长音频: 3分钟×N 分段处理
                       │
                       ▼
               Claude Code 生成结构化笔记
                       │
                       ▼
                  notes/标题.md
```

## 常见问题

### Q: 提示 "yt-dlp 找不到" / "deno 找不到"？
脚本启动时会自动从注册表刷新 PATH。如果仍有问题，重启终端即可。

### Q: 转写结果中文乱码？
模型 `paraformer-zh` 专为中文优化。英文视频尽量走字幕路径（大部分英文视频有自动字幕）。

### Q: 长视频很久没反应？
15 分钟以上的视频 FunASR 会分段处理，每 3 分钟一段，每段约 12 秒。15 分钟视频约需 1.5 分钟转写时间。

### Q: 笔记质量不满意？
编辑 `config.json` 的 `notePrompt`，自定义 prompt 模板。

### Q: 能只下载字幕不生成笔记吗？
带 `-NoCleanup` 运行，从 `notes\_work_*\` 取 `.vtt`/`.srt` 文件。
